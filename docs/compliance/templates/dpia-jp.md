# データ保護影響評価（DPIA）簡易版

> **本テンプレートについて：** GDPR第35条に基づくDPIAの日本語版です。APPI（個人情報保護法）には法的義務としてのDPIAはありませんが、個人情報保護委員会はリスク評価の実施を推奨しています。EU拠点の従業員が対象に含まれる場合は本DPIAの実施が法的義務となります。

## 2. 処理の概要

### 2.1 目的

従業員が業務で使用する外部AIサービス（ChatGPT、Gemini、Claude等）への入力データを自動検査し、個人情報および機密情報の意図しない外部送信を防止する。

### 2.2 処理の法的根拠

| 法域 | 根拠 |
|------|------|
| 日本（APPI） | 第17条（利用目的の特定）に基づく事前通知。正当な事業上の必要性 |
| EU（GDPR） | 第6条1項(f) 正当利益（情報漏洩防止）。従業員の同意は不適切（力の不均衡のため） |

### 2.3 処理の流れ

```
[従業員のブラウザ]
    │ Chrome拡張が対象AIサービスへの入力を検知
    │ 対象ドメイン: chatgpt.com, gemini.google.com, claude.ai
    ▼
[Desktop Agent (localhost:8787)]
    │ テキストの自動検査（正規表現パターンマッチング）
    │ ポリシー評価（ブロック/マスク/警告/許可）
    │ ★ 原文は保存しない
    ▼
[監査ログ (ローカルSQLite)]
    │ 記録内容: ユーザーID、検出カテゴリ、日時、
    │          アクション、SHA256ハッシュ
    │ ★ 原文は記録しない
    ▼
[Pro版のみ: TrustGate Cloud (日本リージョン)]
    統計データ・ハッシュのみ送信
    原文は送信しない
```

### 2.4 取り扱う個人データの種類

| データ種別 | 取り扱い | 保存 | 備考 |
|-----------|---------|------|------|
| 入力テキスト原文 | 検査時に一時的にメモリ上で処理 | **保存しない** | 検査後即時破棄 |
| SHA256ハッシュ | 生成 | 保存する | 原文の復元は不可能 |
| 検出カテゴリ | 生成 | 保存する | 例：「電話番号」「メールアドレス」 |
| ユーザーID | 取得（ヘッダーから） | 保存する | 社内ID |
| 部署 | 取得（ヘッダーから） | 保存する | |
| タイムスタンプ | 生成 | 保存する | |
| 対応アクション | 生成 | 保存する | block/mask/warn/allow |

## 3. リスク評価

### 3.1 リスク一覧

| # | リスク | 発生可能性 | 影響度 | リスクレベル | 軽減策 |
|---|-------|-----------|--------|------------|--------|
| R1 | 従業員のプライバシー侵害（過度な監視と受け取られる） | 中 | 高 | **高** | 対象を外部AIサービスのみに限定。事前通知の徹底。目的外利用の禁止を就業規則に明記 |
| R2 | 検出記録からの個人行動の推測 | 低 | 中 | **中** | 原文非保存。検出カテゴリは「電話番号検出」等の分類のみ。統計利用に限定 |
| R3 | 検出記録の不正アクセス | 低 | 高 | **中** | アクセス権限の厳格な管理。監査ログのアクセスログ記録。暗号化ストレージ |
| R4 | 誤検知による正当な業務の阻害 | 中 | 中 | **中** | Shadowモードによる事前評価。フィードバック機能。ホワイトリスト |
| R5 | 検出記録の人事評価への流用 | 低 | 高 | **中** | 就業規則で目的外利用を明示的に禁止。アクセス権限を情報セキュリティ部門に限定 |
| R6 | 越境データ移転（Pro版） | 低 | 高 | **中** | 日本リージョンのみ使用。原文は送信しない。統計・ハッシュのみ |
| R7 | データの長期保存による不必要な蓄積 | 中 | 低 | **低** | 自動削除（保存期間満了後）。デフォルト90日 |

### 3.2 リスクレベルの定義

| レベル | 定義 | 対応 |
|--------|------|------|
| 高 | 個人の権利に重大な影響を与える可能性がある | 軽減策の実施が必須。実施後に再評価 |
| 中 | 個人の権利に一定の影響を与える可能性がある | 軽減策の実施を推奨 |
| 低 | 個人の権利への影響が限定的 | 現状の管理策で対応可能 |

---

## 4. 軽減策の詳細

### 4.1 技術的措置（TrustGate の設計に組み込み済み）

| 措置 | 説明 | 対応リスク |
|------|------|-----------|
| **原文非保存** | 入力テキストは検査後にメモリから即時破棄。SHA256ハッシュのみ保存 | R1, R2 |
| **ローカル処理** | 検査はPC内のDesktop Agentで完結。外部送信なし | R1, R6 |
| **ドメイン限定** | 指定AIサービスのみ検査。その他のウェブ活動は対象外 | R1 |
| **自動削除** | 保存期間経過後の監査ログの自動パージ | R7 |
| **Shadowモード** | 実施前に検出の影響を評価可能 | R4 |
| **ユーザーID匿名化オプション** | 監査ログのユーザーIDを匿名化する設定が可能 | R2, R5 |

### 4.2 組織的措置

| 措置 | 説明 | 対応リスク |
|------|------|-----------|
| **事前通知** | 全従業員への書面通知（テンプレート提供） | R1 |
| **就業規則改定** | 目的・範囲・従業員の権利を明記 | R1, R5 |
| **アクセス権限管理** | 検出記録へのアクセスを情報セキュリティ部門に限定 | R3, R5 |
| **目的外利用の禁止** | 人事評価・懲戒処分への直接利用を禁止（重大インシデント除く） | R5 |
| **フィードバック制度** | 誤検知の報告・改善要求の窓口を設置 | R4 |
| **定期レビュー** | 四半期ごとのポリシー見直し・誤検知率の評価 | R4 |

## 5. 比例性の評価

### 5.1 必要性

| 質問 | 回答 |
|------|------|
| この処理は目的達成に必要か？ | はい。外部AIサービスへの機密情報入力を防止する他の手段（口頭注意、研修のみ）では実効性が不十分 |
| より侵害性の低い代替手段はないか？ | AIサービスの利用を全面禁止する方法があるが、業務効率への影響が大きすぎる。TrustGateの入力検査は「利用を許可しつつ保護する」最も侵害性の低い方法 |

### 5.2 比例性

| 要素 | 評価 |
|------|------|
| 処理の範囲 | 外部AIサービスへの入力のみ。全ウェブ活動ではない → **比例的** |
| 保存データ | 原文非保存。ハッシュ+メタデータのみ → **最小限** |
| 保存期間 | 90日（デフォルト）→ **合理的** |
| 従業員への影響 | 正当な業務は阻害されない（マスク/ブロックは情報漏洩リスクのある入力のみ）→ **比例的** |
| 透明性 | 事前通知、就業規則記載、問い合わせ窓口あり → **十分** |
