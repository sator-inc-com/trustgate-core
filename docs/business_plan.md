# TrustGate 事業計画書

## 1. エグゼクティブサマリー

TrustGate は **AI Zero Trust Gateway** -- 企業の AI 利用における入出力を リアルタイムで検査・制御するセキュリティプロダクト。

- **課題**: 企業の AI 活用が急速に進む一方、機密情報の漏洩・プロンプトインジェクション等のリスクに対する国産ソリューションが存在しない
- **解決策**: AI の入出力をローカルで検査し、ポリシーに基づいて BLOCK / WARN / MASK / ALLOW を実行するゲートウェイ
- **市場**: AI セキュリティ市場は 2024-2025 年に 5 社が 100-500 億円規模で買収済み。市場の有効性は証明済み
- **ポジション**: 日本市場唯一の国産 AI セキュリティゲートウェイ
- **調達**: ¥30M（Pre ¥300M、希薄化 9%）

### 2.2 日本市場の特殊性

- **個人情報保護法（APPI）** の改正により、AI による個人情報処理への規制強化

- **FISC 安全対策基準**: 金融機関情報システムのセキュリティ基準
- 日本語対応の AI セキュリティ製品が存在しない
- Enterprise 顧客は「国産」「日本リージョン」を強く求める

## 3. プロダクト

### 3.1 プロダクトライン

| | for Applications | for Workforce |
|---|---|---|
| 概要 | 自社 AI システムのサイドカープロキシ | 従業員の AI SaaS 利用を監視・制御 |
| 対象 | RAG チャット、コードアシスタント等 | ChatGPT, Gemini, Claude.ai, Copilot |
| デプロイ | サイドカー Agent | Desktop Agent + ブラウザ拡張 |
| 課金 | per Agent | per seat |
| 購買者 | エンジニア / CTO | 情シス / CISO / 総務 |

### 3.2 技術的優位性

- **完全ローカル処理**: AI 入出力のテキストは顧客環境外に出ない（ハッシュ・統計のみクラウドへ）
- **2 段階検知**: Regex（<5ms）+ Prompt Guard 2 LLM（1-5ms）の 2 段階パイプライン
- **LLM モデルもローカル**: Meta Prompt Guard 2（86M パラメータ）を Agent 内で ONNX 推論。GPU 不要
- **Go 単一バイナリ**: CGO 不要、クロスコンパイル対応（Windows/Linux/macOS）
- **OpenAI 互換 API**: 既存システムの `endpoint` を変えるだけで導入可能

### 3.3 提供形態

| | Free（OSS） | Pro | Enterprise |
|---|---|---|---|
| Agent（検知エンジン） | OSS | 同左 | 同左 |
| Control Plane | なし | SaaS | オンプレミス |
| 管理 UI・レポート | なし | あり | あり + カスタム |
| 監査ログ保存 | メモリ 100 件 | 7 日 | 無制限 |
| SLA | なし | 99.9% | 99.99% + 専任 |

### 8.2 調達の目的

本調達は資金需要ではなく、**リスクヘッジとチャネル獲得**を目的とする。

```
自己資金:   ¥100M（創業者の EXIT 資金）
Year 1-2 赤字: ▲¥35M（最大想定）
→ 自己資金のみでも事業継続可能

調達の真の目的:
  1. 自己資金のリスク分散（消費を ¥20M 以内に抑制）
  2. 投資家（SIer CVC）の販売チャネル活用
  3. 「CVC 出資済み」による Enterprise 顧客の信頼獲得
```
